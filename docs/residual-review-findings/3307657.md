# Black Powder Review Residuals

## P1: Mixed-version clients and server rollback are incompatible

Black Powder adds the `black-powder` and `rainbow-black-powder` rule-set values and the `black` tile-color value to the unversioned socket and persisted-game model. An already-open browser running the previous bundle will omit the black stack and render visible black tiles like hidden tiles. Rolling a server back after a Black Powder game has been persisted will hydrate that game under the previous ascending-only sequence logic.

This does not block the local feature because the app currently has no protocol-capability or deployment-version framework, and adding one would be a separate cross-cutting project. Before production rollout, deploy web and server atomically, force existing browser sessions to reload, and treat persisted Black Powder games as rollback-incompatible. A durable fix should add protocol/capability versioning and reject unsupported persisted rule sets.
